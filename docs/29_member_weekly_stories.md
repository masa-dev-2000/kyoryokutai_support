### P0 実装メモ（2026-06-29）
- **P0-1 月報提出**: `POST /api/monthly-reports`(submit + 承認キュー投入 kind=月次報告) / `monthlyReports.submit` を両 repo に追加 / `mapReport` に bodyMd・planNext / ReportDetailSheet に「AIで本文生成→編集→役場に提出」の導線。提出 → 役場承認 (`decide` が markApproved) まで疎通。
- **P0-2 音声+AI質問**: `VoiceInput`(Web Speech API・クライアント完結・非対応ブラウザは非表示) を活動メモに / `POST /api/ai/followup`(次の一問を返す) を「AIに質問」ボタンに。
- **P0-3 領収書保存**: `POST /api/files`(multipart→Storage put) + `GET /api/files/[...key]`(配信) / Storage 抽象に getBytes 追加 / expenses に `receipt_key` 列(idempotent ALTER) / 活動メモのインライン経費・経費精算の両アップロード点で実保存。

### P1（未実装・説明）
- **P1-1 横断通知レイヤー**: 右上常駐で「差戻し / お知らせ / 承認」を集約。A型のCCメール見落とし防止、B型の申請ステータス追跡が狙い。実装には通知エンティティ(または既存 announcements + approvals + expenses の status 変化を束ねる集約API)と未読管理が要る。MVP外でも効果は大きいが、まず P0 を現場で検証してから。
- **P1-2 役場→隊員コメント（双方向）**: 月報・経費の差戻し/承認時に役場がコメントを返し、隊員側に表示。承認フロー(approvals.detail)にコメント欄を足し、隊員アプリで該当 Report/Expense に紐付け表示する。提出後の「手応えが返らない」問題への直接の打ち手。
- **P1-3 指揮元タグ（団体/役場）**: 活動記録に「この活動は受け入れ団体案件 / 役場案件」のタグを持たせ、B型の宙に浮き問題を可視化。activity_logs に列追加 + フィルタ。承認者を役場1段に保ったまま、将来の団体ロール追加への布石になる。
