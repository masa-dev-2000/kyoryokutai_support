# LLM 比較検討メモ(MVP 選定)

**策定日**: 2026-04-20
**目的**: MVP の AI 処理(月次報告生成 / タグ提案 / 将来の匿名化)にどの LLM を採用するか決定する。
**論点**: Claude 一択で良いのか、それとも別 API(GPT-4o / Gemini / Haiku / 代替)のほうが合理的か。

---

## 1. 使用用途と要求水準

| 用途 | 頻度 | 入力 tok | 出力 tok | 重要な要件 |
|---|---|---|---|---|
| **月次報告生成(F02)** | 月1/人 | 10,000(日報22件) | 3,000 | **日本語要約品質、長文コンテキスト、構造化出力** |
| **タグ自動提案(F14)** | 日1/人 | 500 | 100 | **軽量・高速・安価**、構造化出力 |
| **匿名化処理(F18・Year2)** | 処理毎 | 500 | 500 | **固有名詞検出、抽象化の正確性** |

**要件の優先順位**:
1. 日本語の業務文書品質(自治体提出レベル)
2. コスト(MVP はコスト節約志向)
3. レイテンシ(月次は 45s まで許容、タグは < 3s)
4. プライバシー・データ学習 OFF 可能
5. 日本からの利用安定性

## 3. 用途別の評価

### 3.1 月次報告生成(最重要)

> 日本語の業務文書として、自治体に提出できるレベルの品質が必要。

| 観点 | Sonnet 4.6 | Haiku 4.5 | GPT-5 | GPT-4o | Gemini 2.5 Pro | Gemini 2.5 Flash |
|---|---|---|---|---|---|---|
| 日本語品質 | ◎ | ○+ | ○ | △(直訳調) | ○ | ○- |
| 長文集約(10K tok) | ◎ | ◎ | ◎ | ○ | ◎ | ◎ |
| 冗長回避 | ◎ | ○ | ○ | △ | ○ | ○ |
| コスト(月/人) | $0.80 | $0.25 | $1.60 | $0.50 | $0.55 | $0.10 |
| 総合 | ⭐最強 | ⭐MVPコスパ | 高い | トレードオフ | 悪くない | **激安で実用** |

**用途コスト試算**(隊員1人・月次1回・入力10K/出力3K):
- Opus 4.7: 10K × $15 + 3K × $75 = $0.375 / 月
- Sonnet 4.6: 10K × $3 + 3K × $15 = $0.075 / 月
- **Haiku 4.5: $0.025 / 月** ← MVP に優しい
- **Gemini 2.5 Flash: $0.010 / 月** ← さらに安い

### 3.2 タグ自動提案

> 軽量タスク。頻度が高いので**安価 & 低レイテンシ**が最重要。

| モデル | コスト(日1/人) | レイテンシ | 評価 |
|---|---|---|---|
| Haiku 4.5 | $0.0006 | 〜1s | ⭐ |
| Gemini 2.5 Flash-Lite | $0.0001 | 〜0.5s | ⭐最安 |
| GPT-4o mini | $0.0002 | 〜1s | 優秀 |

→ **タグ提案は Gemini 2.5 Flash-Lite or GPT-4o mini が圧倒的コスト優位**

### 3.3 匿名化処理(Year 2)

固有名詞の検出・抽象化は高品質が必要。推奨は **Sonnet 4.6 or GPT-5**(Year 2 に判断)。

## 4. 非機能比較

| 観点 | Claude | OpenAI | Gemini |
|---|---|---|---|
| データ学習 OFF | 標準で OFF、履歴保持 30日 | 要API設定、保持0日可 | 商用有料版で OFF 可 |
| 日本からの安定性 | ◎ | ◎ | ○(たまにレイテンシ揺れ) |
| プロンプトキャッシュ | ◎(5分/1時間) | ◎(自動) | △(一部) |
| JSON 構造化出力 | ◎(tool_use) | ◎(response_format) | ◎(schema) |
| 長文処理(1M超) | ◎(Sonnet 4.6 / Opus 4.7) | ◎(GPT-5) | ◎(2M) |
| エンタープライズ契約 | ◎ | ◎ | ◎ |
| 個人開発で試しやすい | ◎ | ◎ | ◎ |
| Vercel AI SDK 対応 | ◎ | ◎ | ◎ |

**プライバシー結論**: どれでも自治体提案レベルは満たせる。ただし**エンタープライズ契約/データ保持 0 日化**は Year 2 の県契約時に必須。

## 5. コスト試算(Year 別)

### MVP(隊員10人・1自治体・月次1回+タグ日1)

| 組み合わせ案 | 月次生成($) | タグ($) | 合計/月 |
|---|---|---|---|
| ① Sonnet 4.6(全用途) | $0.80 | $0.07 | **$0.87** |
| ② **Haiku 4.5(全用途)** | $0.25 | $0.02 | **$0.27** |
| ③ **Haiku 4.5 + Gemini Flash-Lite(タグ)** | $0.25 | $0.003 | **$0.25** |
| ④ Gemini 2.5 Flash(全用途) | $0.10 | $0.015 | **$0.12** |
| ⑤ Flash + Sonnet(月次だけ昇格) | $0.80 | $0.015 | **$0.82** |

→ MVP 10人でも月 $1 以下、**どれも許容範囲**。品質リスクを避ける観点で ②③ が合理的。

|---|---|---|
| ② Haiku 4.5 のみ | $27 | $325 |

## 6. モデル抽象化(差し替え可能な設計)

### 推奨: Vercel AI SDK(またはラッパー自作)で抽象化

```ts
// src/lib/ai/provider.ts
import { createAnthropic } from "@ai-sdk/anthropic";
import { createOpenAI } from "@ai-sdk/openai";
import { createGoogleGenerativeAI } from "@ai-sdk/google";

export function getLlm(task: "monthly" | "tags" | "anonymize") {
  switch (task) {
    case "monthly": return anthropic("claude-haiku-4-5");
    case "tags":    return google("gemini-2.5-flash-lite");
    case "anonymize": return anthropic("claude-sonnet-4-6");
  }
}
```

### メリット
- タスク毎に最適モデル
- A/B 品質比較が容易
- ベンダーロックイン回避

## 7. 最終推奨

### MVP 採用案: **② Haiku 4.5 全用途 + 将来差し替え可能な抽象化**

**理由**:
1. Claude の日本語品質は業務文書で定評あり、MVP の品質リスクが最小
2. Haiku 4.5 は Sonnet の 1/3 価格で月次報告も実用水準
3. 単一ベンダーで開発・運用がシンプル(MVP はシンプルさが正義)
4. タスク毎の差し替えは Vercel AI SDK で後から可能

### A/B 検証で試すべきもの(MVP ローンチ後)
- タグ提案のみ **Gemini 2.5 Flash-Lite** へ切替(コスト 1/6、体感品質差なし想定)
- 月次報告の品質不満が出たら **Sonnet 4.6** へ昇格
- 自治体提案時の訴求用に **国産 LLM 比較実験**(Year 2)

---

## 8. 採用した場合のコミット事項

- API キー: Anthropic を MVP 標準に(Claude Haiku 4.5)
- ライブラリ: Vercel AI SDK(`@ai-sdk/anthropic`)でモデルロックイン回避
- データ保持: Anthropic API の `anthropic-beta: prompt-caching` + 学習 OFF 設定
- モデルバージョン: コード側に定数で固定、置換時は明示的に変更
- A/B 実験: Year 1 後半に別 LLM との品質比較ログを残す
