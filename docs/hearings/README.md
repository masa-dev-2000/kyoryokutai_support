# ヒアリング記録

協力隊員・自治体担当・関係者へのヒアリング記録を 1 ファイル / 1 人に分けて保存。

## ファイル命名規則
`YYYY-MM-DD_対象者名.md`
(対象者名は実名 or 仮名。本人の希望に応じて匿名化判断)

## ヘッダー(YAML フロントマター)
```yaml
---
date: YYYY-MM-DD
interviewee: 名前(or 匿名ID)
location: 居住・活動地域
status: 第N回ヒアリング
interviewer: ヒアリング実施者
medium: 対面 / Zoom / 電話 / メール
---
```

## 整理ルール
- ヒアリング内容は加工せず、**生のメモを優先して保存**
- 整理者の解釈・含意は末尾の「メモ(整理者)」に分離
- 機能/UI 提案・戦略議論は **GitHub Issues** に切り出して記録(重複保存しない)
