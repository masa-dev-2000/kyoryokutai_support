### A2. 「役場が楽になる」の "楽" の定義

**質問**: 役場の何が楽になれば「これは買いたい」と思ってもらえる?

**説明**: 同じ "楽" でも以下のように違う。
- **時間**: 月次レビューの工数を 50% 削減
- **頭の使いどころ**: 「考えなきゃいけないこと」が減る(AIが提案)
- **責任分散**: 判断の根拠が AI/データで残る(後で追及されない)
- **見栄え**: 上司・議会・住民への報告物の質が上がる
- **失敗回避**: 離任・トラブルを事前検知

優先順位を 1-5 で。

**[回答]**

```
時間削減: 
頭の使いどころ削減: 
責任分散・記録: 
見栄え向上: 
失敗回避: 

最優先で解きたい "楽" は: ガードレールにより、役場が協力隊にやってほしくないことをやらなくすること。あとは役場側の管理コストを削減するために、協力隊員が自走すること。また、報告用のデータが勝手に集まり、資料が勝手に作成できる状態にすること。
```

## C. バージョン関係

### C1. v2 との関係性

**質問**: v3 は v2 と**独立**? **重畳**? **置き換え**?

**説明**:
| 案 | 説明 | メリット | デメリット |
|---|---|---|---|
| 独立 | v2 知らなくても OK の役場専用アプリ | シンプル、別営業可能 | 隊員データなしで価値が出ない |
| 重畳 | v2 隊員データを活用、v3 は役場ビュー | シナジー強い、隊員 + 役場のセット販売 | v2 導入が前提に |
| 置き換え | v2 を v3 で吸収、隊員 UI は付録扱い | 役場主眼を貫徹 | 隊員モチベ低下リスク |

**[回答]**

```
方針: 置き換え
理由: 役場ライクなものにしつつ、報告などでは隊員データ必須なため。
```
