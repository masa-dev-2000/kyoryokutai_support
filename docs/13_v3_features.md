## 🧱 機能インベントリ(4 カテゴリ・13 機能)

### 🚧 A. ガードレール(役場の判断を肩代わり)

#### A1. 活動範囲ルール集 + 違反検知
- 役場が初期設定:「やってよい・グレー・ダメ」のルール(全国共通テンプレ 80%+カスタム 20%)
- 隊員の行動記録から AI が「グレー / 違反」を自動検知
- アラートは**役場ではなく隊員側に**「これは要確認」と表示
- → 役場が事後で説明できない案件を未然に防ぐ

#### A2. 経費の妥当性チェック(自動仕分け)
- 申請内容を AI がカテゴリ自動仕分け(交通費 / 研修 / 事業 / その他)
- 「過去の承認パターン」と比較して**自動承認候補 / 要確認**を提示
- 役場は **要確認のみ判断**(月数件まで削減)

#### A3. 役場との確認要否 AI 判定
- 隊員が「これやっていい?」と聞きたい時、まず AI が回答
- AI が「OK / グレー / 役場確認推奨 / ダメ」を判定
- 「役場確認推奨」のみ役場に通知

---

### 🚶 B. 自走支援(隊員の自律性を引き出す)

#### B1. 「これやっていい?」FAQ(過去事例から即答)
- 隊員の質問に AI が過去事例 + ガードレール ルールから即答
- 例:「空き家所有者にお金を渡してもいい?」→ NG 事例提示
- 例:「自治体外で活動していい?」→ 範囲ルール提示
- → 役場に聞かなくても判断できる

#### B2. 報告書ドラフト自動化(豊岡 kintone 機能の延長)
- 行動記録(音声・テキスト)から AI が月次報告下書き作成
- 隊員は確認・編集して提出
- 「Word を埋め込む」豊岡方式より、AI ベースで進化

#### B3. 判断補助カード(信頼スコア付き)
- 過去の事例 + ガードレールルール + 自治体の固有事情から
- 「やってよい(★★★)/ 微妙(★★)/ 役場に確認(★)/ ダメ」
- AI は最小限の補助:**判定根拠を隊員に提示**(納得感)

---

### 📊 C. 自動レポート(役場に勝手に届く)

#### C1. 月 1 メールダイジェスト(役場用)
- 担当隊員の活動サマリを月 1 回メール配信
- 「アラート 0 件・経費 OK・進捗順調」のような要約
- **これだけで役場は把握完了**(週 1 回開く必要すらなくす)

#### C2. 議会報告テンプレ自動生成
- 月次・四半期・年次の議会用 Word ファイル自動生成
- 「Word ファイル添付定番」の役場文化に合わせる
- 数字・グラフ・隊員写真 + 引用

#### C3. 県・国 報告フォーマット自動入力
- 県の協力隊実績報告フォーマットに自動入力
- 特別交付税申請の根拠資料も自動作成
- 「報告のための資料作成」工数ゼロを目指す

---

### 📋 D. ベース業務機能(豊岡 kintone 最低限・必須)

> オーナー指示: 「豊岡の kintone システムの機能は最低限必要」

#### D1. 経費申請フロー(豊岡風)
- 隊員がレシート画像撮影 → 申請完了
- 役場は申請一覧で承認 / 差戻し(A2 で自動仕分け済)
- 経費執行履歴・予算残高表示

#### D2. 月次報告 提出フロー
- 隊員: 月次レポートを提出
- 役場: レビュー + 承認(評価コメント必須・Issue #10 継承)
- 承認済は議会報告(C2)にそのまま引用

#### D3. 役場 → 隊員 お知らせ配信
- 役場がお知らせを作成・配信
- 隊員側で既読管理
- (隊員 → 役場のチャットは外す。電話・LINE 運用尊重)

#### D4. 厳格な権限分離(豊岡弱点の逆張り)
- 隊員は自分の情報のみ閲覧可
- 役場担当者は担当隊員のみ
- → 「他隊員の情報も見える」豊岡弱点を構造的に解消

## 🎨 UX 設計指針

### 役場側 UI
- **シンプル + 業務系**(役場の信頼感、Excel 慣れ層と齟齬なし)
- 週 1 回メールで足りる設計
- ダッシュボードは「アラートがある時だけ赤くなる」
- 操作不要時は**通知ゼロ**

### 隊員側 UI
- v2 で培ったバブル UI を**部分流用 OK**
- ただし「ガードレール内の自走」が伝わる設計が核
- 判断補助カードが目立つ位置
- 「楽だが監視されてない感」を演出

### デバイス
- 役場: PC(Windows・Edge 想定)
- 隊員: スマホ(現場入力)

### 外部ツール連携
- **Word / Excel エクスポート必須**(議会報告)
- LGWAN・自治体クラウド要件は調査(G2 オーナー回答「勉強が必要」)

## 💰 価格モデル(オーナー回答 D 案 = 既定路線)

```
9,800 円/月/隊員
```

### 試算
| 自治体規模 | 隊員数 | 月額 | 年額 |
|---|---|---|---|
| 小規模(新温泉町等) | 3 名 | 29,400 円 | 約 35 万円 |
| 中規模(佐用町) | 4-6 名 | 49,000 円 | 約 59 万円 |
| 大規模(豊岡市) | 30 名 | 294,000 円 | 約 353 万円 |
| 巨大(海士町) | 163 名 | 1,597,400 円 | 約 1,917 万円 |

### 年商見通し(3 年目)
- 50 自治体 × 平均 6.7 名 = 335 名 × 9,800 × 12 = **約 3,940 万円**
- 100 自治体到達なら 約 7,880 万円
- 130 自治体到達なら 1 億超

→ オーナー指示「目標下げる」= **3 年で 50 自治体・年商 4,000 万円**が現実線

### 営業上の合理性
- 1 自治体 35-59 万円は **随契枠(委託 200 万円)に余裕で収まる**
- 議会承認不要、課長決裁レベルで通る金額
- 大規模自治体は議会案件になる可能性

## ❌ やらないこと(B3 + 私の追加)

| やらない | 理由 |
|---|---|
| AI のリッチ機能(壁打ち、戦略レビュー等) | オーナー指示・AI は最小限の補助のみ |
| 隊員の要望を全部取り入れること | 役場主眼を貫徹 |
| 隊員間チャット・SNS | 余計、Slack/LINE 競合 |
| 詳細業務管理(タイムカード等) | 「楽になる」と逆行 |
| 県共同調達向け機能(現時点) | まずは個別自治体販売 |
| SSO・LGWAN 統合(初期版) | 自治体別対応コスト |
| 全国事例検索 DB(v2 の MUST だった) | 機能リッチ化を避けるため後回し |
