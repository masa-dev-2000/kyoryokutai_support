## 🥇 競合との根本的な差別化(最重要)

### 既存日報 SaaS / 業務管理 SaaS の暗黙の前提

| カテゴリ | 代表 | 暗黙の前提 |
|---|---|---|
| 日報 SaaS | gamba! / Teamspirit / ai日報 | **管理職が積極的に読み・コメントする** |
| プロジェクト管理 | Asana / Trello / Notion | **管理職が割り振り・進捗確認する** |
| コミュニケーション | Slack / Teams | **常時双方向のやり取り** |
| 業務管理 | kintone | **設計者・管理者が運用する** |
| 評価 SaaS | カオナビ等 | **評価者・人事の関与** |

### v3 の根本的な思想転換

> **管理職が動かなくても、組織が動く。**

- 隊員(従業員)は **ガードレール内で自走** する
- 役場(管理職)は **週 1 回見るだけ**
- AI と仕組みが **マネジメントの一部を構造的に肩代わり** する

→ これは既存カテゴリのどれにも属さない、**新カテゴリ "自走型組織 SaaS"** の可能性。

### 訴求の使い分け(3 段階の言い換え)

| 場面 | 表現 |
|---|---|
| 内部設計思想 | 「管理職が動かなくても回る」 |
| 役場・経営者ピッチ | **「管理職の手間を最小限に」**「最小限の関与で済む」 |
| 議会・対外説明 | 「省力化された管理」「自走型組織」 |

→ 同じ思想を **聞き手に応じて言い換える** 訴求設計。

## 🧱 機能インベントリ(4 カテゴリ・13 機能)

### 🚧 A. ガードレール(役場の判断を肩代わり)

#### A1. 活動範囲ルール集 + 違反検知
- 役場が初期設定:「やってよい・グレー・ダメ」のルール(全国共通テンプレ 80%+カスタム 20%)
- 隊員の行動記録から AI が「グレー / 違反」を自動検知
- アラートは**役場ではなく隊員側に**「これは要確認」と表示
- → 役場が事後で説明できない案件を未然に防ぐ

#### A2. 経費の妥当性チェック(自動仕分け)
- 申請内容を AI がカテゴリ自動仕分け(交通費 / 研修 / 事業 / その他)
- 「過去の承認パターン」と比較して**自動承認候補 / 要確認**を提示
- 役場は **要確認のみ判断**(月数件まで削減)

#### A3. 役場との確認要否 AI 判定
- 隊員が「これやっていい?」と聞きたい時、まず AI が回答
- AI が「OK / グレー / 役場確認推奨 / ダメ」を判定
- 「役場確認推奨」のみ役場に通知

---

### 🚶 B. 自走支援(隊員の自律性を引き出す)

#### B1. 「これやっていい?」FAQ(過去事例から即答)
- 隊員の質問に AI が過去事例 + ガードレール ルールから即答
- 例:「空き家所有者にお金を渡してもいい?」→ NG 事例提示
- 例:「自治体外で活動していい?」→ 範囲ルール提示
- → 役場に聞かなくても判断できる

#### B2. 報告書ドラフト自動化(豊岡 kintone 機能の延長)
- 行動記録(音声・テキスト)から AI が月次報告下書き作成
- 隊員は確認・編集して提出
- 「Word を埋め込む」豊岡方式より、AI ベースで進化

#### B3. 判断補助カード(信頼スコア付き)
- 過去の事例 + ガードレールルール + 自治体の固有事情から
- 「やってよい(★★★)/ 微妙(★★)/ 役場に確認(★)/ ダメ」
- AI は最小限の補助:**判定根拠を隊員に提示**(納得感)

---

### 📊 C. 自動レポート(役場に勝手に届く)

#### C1. 月 1 メールダイジェスト(役場用)
- 担当隊員の活動サマリを月 1 回メール配信
- 「アラート 0 件・経費 OK・進捗順調」のような要約
- **これだけで役場は把握完了**(週 1 回開く必要すらなくす)

#### C2. 議会報告テンプレ自動生成
- 月次・四半期・年次の議会用 Word ファイル自動生成
- 「Word ファイル添付定番」の役場文化に合わせる
- 数字・グラフ・隊員写真 + 引用

#### C3. 県・国 報告フォーマット自動入力
- 県の協力隊実績報告フォーマットに自動入力
- 特別交付税申請の根拠資料も自動作成
- 「報告のための資料作成」工数ゼロを目指す

---

### 📋 D. ベース業務機能(豊岡 kintone 最低限・必須)

> オーナー指示: 「豊岡の kintone システムの機能は最低限必要」

#### D1. 経費申請フロー(豊岡風)
- 隊員がレシート画像撮影 → 申請完了
- 役場は申請一覧で承認 / 差戻し(A2 で自動仕分け済)
- 経費執行履歴・予算残高表示

#### D2. 月次報告 提出フロー
- 隊員: 月次レポートを提出
- 役場: レビュー + 承認(評価コメント必須・Issue #10 継承)
- 承認済は議会報告(C2)にそのまま引用

#### D3. 役場 → 隊員 お知らせ配信
- 役場がお知らせを作成・配信
- 隊員側で既読管理
- (隊員 → 役場のチャットは外す。電話・LINE 運用尊重)

#### D4. 厳格な権限分離(豊岡弱点の逆張り)
- 隊員は自分の情報のみ閲覧可
- 役場担当者は担当隊員のみ
- → 「他隊員の情報も見える」豊岡弱点を構造的に解消

### 試算(オーナー指示で大規模を除外)

> 豊岡市・海士町など大規模自治体は **既に管理システム導入済の可能性が高い** ため、
> 試算と営業ターゲットから外す。

| 自治体規模 | 隊員数 | 月額 | 年額 | ターゲット? |
|---|---|---|---|---|
| 小規模(新温泉町等) | 3 名 | 29,400 円 | 約 35 万円 | ✅ 主戦場 |
| 中規模(佐用町等) | 4-7 名 | 約 54,000 円 | 約 65 万円 | ✅ ターゲット |
| ~~大規模(豊岡市等)~~ | ~~30 名~~ | - | - | 🚫 既存システムあり |
| ~~巨大(海士町等)~~ | ~~100+ 名~~ | - | - | 🚫 既存システムあり |

### 年商見通し(3 年目・現実線)
- 50 自治体 × 平均 5.5 名 = 275 名 × 9,800 × 12 = **約 3,234 万円**
- 100 自治体到達(全国の 8.5%)で 約 6,468 万円
- 130 自治体到達で 約 8,400 万円
- **150 自治体到達(全国の 13%)で 約 1 億円**

→ **3 年で 50 自治体・年商 約 3,200 万円**が現実線、5 年で年商 1 億超を目指す軌道。

### 営業上の合理性
- 1 自治体 35-65 万円は **随契枠(委託 200 万円)に余裕で収まる**
- 議会承認不要、課長決裁レベルで通る金額
- 「未導入の小規模自治体」が圧倒的多数 = 営業ターゲット数は確保できる

### 設計指針(初期実装から意識する)

#### A. データモデルの抽象化

| 協力隊文脈 | 抽象化後 | 一般組織展開例 |
|---|---|---|
| 協力隊員 | **Member** | 営業員 / 現場作業員 / リモートワーカー |
| 自治体 | **Organization** | 中小企業 / NPO / 第三セクター |
| 役場担当者 | **Manager** | 部課長 / 現場監督 / HR |
| 県(親組織) | **ParentOrg** | 本社 / 業界団体 / 本部 |
| 議会報告 | **ExternalReport** | 株主・取締役会向け / 業界団体向け |
| 県報告 | **ParentOrgReport** | 本社報告 |
| 活動費 | **Budget** | 予算 / 経費枠 |

#### B. ガードレール機能の業種別テンプレ

```
common/         共通ルール(法令・倫理)
templates/
├── chiiki/     協力隊向け(MVP)
├── sales/      営業職向け(Year 3+)
├── construction/ 建設現場向け(Year 3+)
├── care/       介護・医療向け(Year 3+)
└── npo/        NPO 向け(Year 3+)
```

#### C. 実装上の注意点

- ✅ コンポーネント名は汎用名(`MemberCard`, `OrgDashboard`, `GuardrailRule` 等)
- ✅ DB テーブル名は汎用(`members`, `organizations`, `guardrails`)
- ✅ ガードレールルールは YAML / JSON で差し替え可能
- ✅ レポートテンプレも業種別ディレクトリ構造
- ⚠️ ただし **過度な汎用化は禁物**(YAGNI)
- ⚠️ コピー(画面文言)は **協力隊版を MVP として優先**、汎用化は Year 3 で実施

### 刺さる業界マップ(将来の汎用展開)

「管理職が動かなくても回る」が特に刺さる組織:

| 業界 | 刺さる理由 |
|---|---|
| **協力隊管理(MVP)** | 役場担当者が異動頻繁、管理経験浅い |
| 建設現場(Year 3+) | 現場監督が複数現場掛け持ち、物理的不在 |
| 介護施設(Year 3+) | 管理職が現場業務で忙しい、シフト型 |
| 第三セクター・NPO(Year 3+) | 管理職のマネジメント力が薄い |
| 中小企業 営業職(Year 3+) | 外勤多、社長が現場見れない |
| リモートワーク企業(Year 3+) | 管理職が物理的に近くにいない |
| 副業・複業経営者(Year 3+) | 経営者が複数事業を抱える、時間が分散 |

→ 共通点: **「管理する側のリソースが構造的に足りない」** 組織。日本社会全体で増えている層。

### Partner 警告: 一般展開の落とし穴

- 「協力隊で動いたから一般組織でもいける」と思いがち
- 実は **カスタマーサクセスのコスト構造が全く違う**(自治体 vs 中小企業)
- 一般展開時は **営業・サポート体制の再構築が必要**
- → Plan B でも **「Year 3 までは協力隊一筋」** が安全策
